Обзор

SQL Injection Toolkit - это продвинутый инструмент для тестирования веб-приложений на уязвимости SQL-инъекций.

Основные возможности:
  • Обнаружение 4 типов SQL-инъекций
  • Поддержка 5 типов СУБД
  • Автоматическая эксплуатация уязвимостей
  • Мутация payload'ов для обхода WAF
  • Многопоточное сканирование
  • Веб-интерфейс с визуализацией результатов

Использование

Базовое сканирование:
  1. Введите целевой URL в поле "Целевой URL"
  2. Выберите уровень сканирования и риск
  3. Нажмите "Начать сканирование"
  4. Следите за прогрессом в реальном времени
  5. Изучите результаты после завершения
Параметры:
  • Целевой URL: Адрес страницы для тестирования
  • POST данные: Данные для POST запросов (param1=value1¶m2=value2)
  • Cookies: Cookie строка для аутентификации
  • Уровень сканирования: Глубина тестирования (1-5)
  • Уровень риска: Опасность payload'ов (1-3)
  • Потоки: Количество одновременных запросов
  • Таймаут: Время ожидания ответа
  • Мутация: Включить обход WAF

Уровни сканирования

Уровень 1 - Базовый:

Тестирование простыми payload'ами (кавычки, слэши)

  • '
  • "
  • \
  • ' OR '1'='1
Уровень 2 - Union-based:

Тестирование UNION инъекций

  • ' UNION SELECT NULL--
  • ' UNION SELECT 1,2,3--
  • ' UNION SELECT @@version--
Уровень 3 - Error-based:

Вызов SQL ошибок для извлечения данных

  • EXTRACTVALUE()
  • FLOOR(RAND())
  • COUNT() GROUP BY
Уровень 4 - Boolean-based:

Логические инъекции с истинными/ложными условиями

  • ' AND 1=1--
  • ' AND 1=2--
  • SUBSTRING() проверки
Уровень 5 - Time-based:

Инъекции с временной задержкой

  • SLEEP()
  • BENCHMARK()
  • Heavy JOIN запросы

Мутация payload'ов

Мутация позволяет обходить WAF (Web Application Firewall) защиты.

Техники мутации:
  • Case Variation: Случайный регистр символов
  • Encoding: URL, HEX, Double URL кодирование
  • Comments: Вставка комментариев в SQL
  • Whitespace: Изменение пробелов и табов
  • Concatenation: Использование CONCAT функций
  • Character Encoding: CHAR() функции
Пример мутации:
Оригинал: ' UNION SELECT @@version--
Мутация 1: ' UnIoN SeLeCt @@VeRsIoN--
Мутация 2: '/**/UNION/**/SELECT/**/@@version--
Мутация 3: ' UNION%20SELECT%20@@version--

Примеры использования

Пример 1: Базовое сканирование GET параметра
URL: http://example.com/news.php?id=1
Уровень: 2
Результат: Найдена Union-based инъекция
Пример 2: Сканирование POST формы
URL: http://example.com/login.php
POST: username=admin&password=test
Уровень: 3
Результат: Найдена Error-based инъекция в username
Пример 3: Обход WAF
URL: http://example.com/search.php
Мутация: Включена
Уровень: 5
Результат: Обойден WAF, найдена Time-based инъекция
Тестовые URL для практики:
  • http://localhost:8080/test?id=1 - Тестовый сервер
  • http://testphp.vulnweb.com/artists.php?artist=1 - Acunetix тест
  • http://testfire.net/bank/login.aspx - IBM тест
Быстрые советы
✅ Рекомендуется:
  • Начинайте с низких уровней сканирования
  • Используйте мутацию при обнаружении WAF
  • Проверяйте результаты вручную
  • Сохраняйте историю сканирований
⚠️ Осторожно:
  • Не используйте на продакшн сайтах без разрешения
  • Высокие уровни риска могут вызвать DoS
  • Time-based атаки медленные
  • Соблюдайте законодательство
